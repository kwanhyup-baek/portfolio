# Lessons Learned

## Review feedback is system data

Reviewer comments can improve future work when converted into reusable rules.

## The reviewer-author gap is a design problem

Explicit quality expectations reduce mismatched interpretations.

## LLM context should be compact

Short, relevant notes reduce repeated loading of large reference materials.

## Immediate LGTM is more useful than eventual approval

It measures how often a submission is ready for review.

## The knowledge system must evolve

New feedback, failures, and successful patterns should update the guidance.
